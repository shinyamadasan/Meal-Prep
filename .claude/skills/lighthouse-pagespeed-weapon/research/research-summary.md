## 5 most influential sources

### 1. `external/2026-05-20-lighthouse-performance-scoring.md`
**Why:** Answers the command brief's explicit open question — "Does Lighthouse v12 change Performance
score weighting for INP?" Answer: No. Lighthouse 10 and v12 use identical weights (TBT 30%, LCP 25%,
CLS 25%, FCP 10%, SI 10%). INP does NOT appear in the Lighthouse Performance score. TBT remains the
lab proxy. TTI was removed in Lighthouse 10. This is the foundational fact that `guides/06-audit-category-glossary.md`
and `guides/02-lab-vs-field.md` must both state clearly.

### 2. `external/2026-05-20-lab-vs-field-data-differences.md`
**Why:** The canonical Google source for the conceptual framework behind `guides/02-lab-vs-field.md`.
Covers all divergence causes per metric, the p75 measurement convention, and the critical TBT/INP
limitation. Key quotation to carry forward: "a page may have poor INP despite having very good TBT
scores." This is the most important single fact in the whole weapon's conceptual foundation.

### 3. `external/2026-05-20-lhci-github-actions-guide.md`
**Why:** Covers the complete LHCI GitHub Actions integration with exact YAML, gotchas (fetch-depth),
and both authentication paths. Documents that LHCI 0.15.x uses Lighthouse 12.6.1, Node 18+ is
required, and the PWA category was removed in Lighthouse 12 (May 2024). This drives
`guides/03-ci-integration.md` and `templates/lighthouserc-starter.yaml`.

### 4. `external/2026-05-20-lhci-budget-assertions.md`
**Why:** Provides the complete `assert` block reference including both threshold types (`minScore` vs
`maxNumericValue`), aggregation methods (`median-run` vs `pessimistic`), and the practice of measuring
baseline before setting budgets. Directly informs `guides/01-lhci-configuration.md` and the
`lighthouserc` starter template. Encodes the critical directive: never gate CI on a budget lower than
current production baseline.

### 5. `external/2026-05-20-psi-crux-integration.md`
**Why:** Official tool comparison table showing exactly how PSI, CrUX API, CrUX History API, GSC, and
BigQuery differ in update frequency, metric coverage, and data granularity. Explains the two-block PSI
API response structure (`loadingExperience` for field data, `lighthouseResult` for lab data). This is
the reference structure for `guides/02-lab-vs-field.md`'s "which tool for which job" section.

## Key facts for weapon-forge to encode

- Lighthouse 12.6.1 is current (LHCI 0.15.x). PWA category removed in Lighthouse 12. Four categories remain: Performance, Accessibility, Best Practices, SEO.
- Performance score weights (LH10+/LH12): TBT 30%, LCP 25%, CLS 25%, FCP 10%, Speed Index 10%.
- INP replaced FID as Core Web Vital on March 12, 2024. TBT is the lab proxy for INP (imperfect correlation).
- FID removed from CrUX tools on September 9, 2024. Do not reference FID in new LHCI assertions.
- CrUX Dashboard deprecated November 2025. CrUX Vis or BigQuery are the replacements.
- SpeedCurve acquired by Embrace in November 2025. Pricing: $90/$576/$1,680 per month.
- PSI API response: `loadingExperience` (CrUX field) + `lighthouseResult` (lab). URL-level falls back to origin-level when insufficient CrUX data.
- April 2026: 56.4% of origins pass all three Core Web Vitals (LCP + INP + CLS).
- Default Lighthouse mobile throttling: 1.6 Mbps bandwidth, 150 ms RTT, 4x CPU. This simulates the slowest 5-10% of user experiences.
- LHCI requires `fetch-depth: 20` in GitHub Actions checkout to avoid ancestor detection errors.
- Plugin API constraint: plugins cannot use custom Gatherers. Custom Gatherers require full custom Lighthouse config.
